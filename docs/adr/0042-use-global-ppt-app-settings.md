# PPT App 使用单一全局设置来源

PPT App 的生成偏好统一由工作区根目录下的全局 `setting.json` 持有。新建和已有 Workspace 都读取这份全局设置；修改设置时也只更新全局文件。

Workspace 不再创建自己的 `setting.json`。历史 Workspace 中已经存在的 `setting.json` 保留但不再读取、更新或自动清理。正在运行的生成任务不因设置修改而中断，后续任务和后续读取使用最新全局设置。

**考虑过的方案**

- 同时维护全局默认值和 Workspace 设置。放弃，因为用户难以判断当前修改影响一个项目还是所有项目，已有实现也会同时写入两处，形成不清晰的权威来源。
- 创建 Workspace 时复制全局设置快照。放弃，因为历史项目会持续使用旧偏好，和 Settings 作为应用级入口的产品语义冲突。

**结果**

- Settings 是应用级偏好入口，不依赖当前是否打开 Workspace。
- 所有项目共享页面生成并发、视觉检查和研究搜索控制等生成偏好。
- Workspace 仍持有演示需求、大纲、页面与生成产物等项目内容，但不持有应用偏好。
- 兼容接口可以继续接受 Workspace 路径，但其设置写入语义必须落到全局设置。
