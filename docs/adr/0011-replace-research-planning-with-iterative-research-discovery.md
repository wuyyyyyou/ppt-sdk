# Replace Research Planning with iterative Research Discovery

Deck Generation will retire the one-shot, per-page `Research Plan` decision model and use deck-level iterative Research Discovery instead. Research Discovery runs web discovery before visual discovery, uses LLM JSON decisions to choose whether and what to search next, uses Agent sessions only for curation, stores curated material in a Research Discovery Evidence Pool, and then updates the Page Plan with evidence-aware `content_plan` references before code deterministically materializes page-assigned Research Evidence. This trades implementation simplicity for a closed research loop that can discover missing facts and image needs from already curated material while keeping Page Authoring grounded only in assigned Research Evidence.

**Considered Options**

- Keep the existing `ResearchPlan -> per-page collect/curate` flow. Rejected because it asks the LLM to predict every page's evidence needs before seeing any external material, which causes missed searches, premature page-level boundaries, and weak image decisions.
- Let Page Authoring search when it needs more facts or images. Rejected because Page Authoring must not use Raw Research Material or ad-hoc search as grounding evidence.
- Put all discovered material directly into Shared Research Evidence. Rejected because then every page could ground itself on the whole deck's evidence pool, bypassing page-level evidence assignment.
- Create a separate Page Content Plan artifact. Rejected because Page Plan is already the page-planning boundary; Evidence-Aware Page Planning should update Page Plan entries with `content_plan` rather than introduce a second competing page plan.

**Consequences**

- `research-plan.json` becomes a legacy artifact and is no longer a new-flow decision input.
- Research Discovery is non-blocking: reaching an iteration limit records Research Evidence Gaps and generation continues with omitted, generalized, or TBD / 待补充 content.
- Refinement uses the same Research Discovery Loop with a target scope; it updates only the targeted Page Plan entries and corresponding page-assigned Research Evidence.
- Page-assigned Research Evidence is generated by code from Page Plan evidence references, not written directly by an LLM or Agent.
