# Agent Freestyle 页面创作说明

`BlankSlide.tsx` 只用于模板发现、预览和 fork 后的技术起点。创建具体演示时，由 Agent 根据当前讨论新建或重写最终 `slides/*.tsx`。

## 创作原则

- 业务内容、文案和页面结构可以直接写在当前页 TSX 中。
- 页面内容直接写入 TSX，不需要创建对应的数据 JSON。
- 不使用蓝图，不要求先选择布局族，也不以组件复用率为质量目标。
- 页面必须兑现逐页文字设计中的核心表达、视觉焦点、构图和禁止项。
- 不得因为地图、时间线、信息图或自由排版实现较难，就退化成通用卡片列表。
- 只有跨页重复且视觉语义相同的元素才适合抽成共享组件。

## 渲染约束

- 页面固定为 `1280x720`，不得滚动或依赖交互。
- 关键标题、正文、数字和标签应尽量保留为真实 DOM 文本。
- 复杂图表或图形区域使用 DOM、SVG 或普通图片表达，其外围说明文字保持为 DOM 文本。
- 本地图片使用渲染器可访问的稳定 URL，不使用 data URL。
- 不用伪元素承载关键内容；谨慎使用依赖浏览器合成的滤镜、混合模式和复杂 transform。
- 每个最终页面只需默认导出一个 React 组件。页面身份和名称由 `manifest.json` 提供。
